The Review Process

Common Issues Found
in App Reviews

Across 600,000+ app reviews over six years, three categories of UX problems appeared most consistently — each solvable with disciplined application of Apple's Human Interface Guidelines.

01 — Navigation
Navigation Misalignment

Developers often implement custom navigation patterns that conflict with native iOS behaviors, creating disorientation for users accustomed to platform conventions.

Non-standard tab nav Hidden primary actions Gesture conflicts

"Navigation should be predictable and consistent with platform conventions."

  • Use standard UITabBar components
  • Preserve system gesture behaviors
  • Maintain consistent navigation hierarchy
02 — Hierarchy
Visual Hierarchy Problems

Many apps fail to establish clear hierarchy, making interfaces difficult to scan. Competing visual elements fragment attention and slow task completion.

Excessive type styles Poor spacing Lack of grouping

"Interfaces should communicate hierarchy clearly through typography, spacing, and color."

  • Apply SF typography scale
  • Use 8-point spacing system
  • Reduce competing visual elements
03 — Feedback
Interaction Feedback

Users often receive insufficient feedback for actions, leaving them uncertain whether an interaction registered — a critical trust failure in consumer apps.

No button states Missing loading indicators No haptic feedback

"People should always understand the result of their actions."

  • Implement visual state changes
  • Add micro-animations on interaction
  • Use system haptic feedback patterns

Key Takeaways

What Makes Apple Platform
Design Succeed

I · HIG
Follow the HIG
Human Interface Guidelines exist because Apple has studied how people use their devices for decades. Working with the platform — not against it — always wins.
II · Convention
Respect Conventions
Users build muscle memory with native patterns. Deviating from platform conventions creates friction, increases cognitive load, and erodes trust.
III · Feedback
Prioritize Feedback
Every action a user takes deserves a response. State changes, haptics, and micro-animations are not decoration — they are the grammar of a trustworthy interface.
IV · Access
Design for Accessibility
Accessibility is not a checklist item added at the end. Apps designed with Dynamic Type, sufficient contrast, and reachable targets are better for every user.
